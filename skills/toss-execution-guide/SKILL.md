# Toss Execution Guide

Use this skill when the user wants an easy-to-scan execution guide rather than a plain summary.

## Workflow

1. Read the provided requirements and relevant local code.
2. Separate confirmed facts, inferred structure, and open questions.
3. Reconstruct the topic as an execution guide.
4. Output a Toss-style single HTML file.

## Required Sections

- one-line summary
- why this matters
- key points
- scope / non-scope
- AS-IS / TO-BE
- execution plan
- data / API / settings contract
- class diagram
- sequence diagram
- flowchart
- implementation checklist
- QA checklist
- ops checklist
- risks and open decisions
- remaining work
