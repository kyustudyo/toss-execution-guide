어떤 기능, 정책, SDK, 구조, 제품 요구사항을 주더라도 토스 스타일의 단일 HTML 실행 가이드를 작성해줘.

요구사항:
- 단순 요약이 아니라 실행 가이드여야 함
- 한 줄 요약, 범위/비범위, AS-IS/TO-BE, 실행 계획, 체크리스트 포함
- 클래스 다이어그램, 시퀀스 다이어그램, 플로우차트 포함
- 실제 연결선, 화살표, 분기, 합류가 보이도록 작성
- 다이어그램이 크면 `+` 버튼으로 팝업에서 한 화면에 맞춰 볼 수 있게 구성
- 확인된 사실 / 추정 / 미확정 분리
- 긴 URL을 raw text로 노출하지 말고 짧은 링크 라벨 사용
- HTML은 밝은 배경, 흰 카드, 파란 포인트, 큰 타이포의 토스 스타일
